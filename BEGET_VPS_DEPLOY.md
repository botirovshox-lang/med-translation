# Развёртывание Medical CAT Translator на Beget VPS

Полный путь от заказа сервера до работающего сайта на вашем домене.
Общее время: **~40 минут**, из них активных действий — около 15.

---

## 📋 Что вам нужно перед стартом

- ✅ Аккаунт на Beget (он уже есть)
- ✅ Домен (если ещё нет — закажите в разделе **Домены** у Beget или используйте поддомен от вашего существующего домена, например `med.вашсайт.ru`)
- ✅ Карта для оплаты VPS

---

## Шаг 1. Заказ VPS (5 минут)

1. Beget Cloud → раздел **Серверы** → **Создать**
2. Параметры:
   - **Регион:** Санкт-Петербург (низкий пинг по России)
   - **ОС:** Ubuntu 24.04
   - **Тариф:** **810₽** (2 ядра / 2 ГБ / 30 ГБ NVMe) или **990₽** если хочется RAM с запасом
   - **Публичный IPv4:** ✅ обязательно (+150₽)
3. Нажмите **Создать**
4. Beget пришлёт на email:
   - **IP-адрес сервера** (например `194.87.X.X`)
   - **SSH-логин и пароль** (root)

📌 Сохраните это, понадобится через минуту.

---

## Шаг 2. Привязка домена (5 минут)

В разделе Beget → **DNS** добавьте A-запись:

| Тип | Поддомен | Значение | TTL |
|---|---|---|---|
| A | `med` (или `@` если хотите на основной домен) | IP с предыдущего шага | 300 |

После сохранения DNS обновится за **5–20 минут**. Проверить:
```bash
nslookup med.вашдомен.ru
```
Должен вернуть IP вашего VPS.

---

## Шаг 3. Подключение к серверу по SSH (1 минута)

Откройте PowerShell или Windows Terminal:

```bash
ssh root@194.87.X.X
```
(IP — тот, что прислал Beget). Введите пароль из письма.

При первом подключении система попросит сменить пароль — задайте новый надёжный пароль и запишите его в надёжное место.

---

## Шаг 4. Автоматическая установка (10 минут)

Скопируйте и вставьте **одной строкой** прямо в SSH-консоль сервера:

```bash
curl -fsSL https://raw.githubusercontent.com/botirovshox-lang/med-translation/main/deploy/setup_vps.sh | sudo bash
```

Скрипт сделает **всё сам**:

| № | Что делает |
|---|---|
| 1 | Обновит систему |
| 2 | Установит Python 3, Nginx, Certbot, build-tools |
| 3 | Включит firewall (открыты только SSH и HTTPS) |
| 4 | Создаст отдельного пользователя `medcat` для приложения |
| 5 | Склонирует репозиторий в `/opt/med-translation` |
| 6 | Установит все зависимости в виртуальное окружение |
| 7 | Настроит `systemd` сервис (автозапуск, перезапуск при падении) |
| 8 | Настроит Nginx и выпустит **бесплатный SSL** через Let's Encrypt |

В конце скрипт спросит:

1. **Пароль для входа в приложение** — придумайте свой
2. **Ваш домен** — введите `med.вашдомен.ru` (тот же, что прописали в DNS)

После завершения откройте в браузере **https://med.вашдомен.ru** — приложение работает.

---

## Шаг 5. (Опционально) Добавить API-ключи

Если хотите **реальные** переводы через OpenAI/Google вместо демо-заглушек:

```bash
sudo nano /etc/medcat/env
```

Добавьте строки:
```
OPENAI_API_KEY=sk-...
GOOGLE_TRANSLATE_API_KEY=ya29...
ANTHROPIC_API_KEY=sk-ant-...
```

Сохраните (Ctrl+O, Enter, Ctrl+X) и перезапустите:
```bash
sudo systemctl restart medcat
```

## 🔧 Полезные команды для управления

| Команда | Что делает |
|---|---|
| `sudo systemctl status medcat` | Статус приложения |
| `sudo systemctl restart medcat` | Перезапуск |
| `sudo journalctl -u medcat -f` | Просмотр логов в реальном времени |
| `cd /opt/med-translation && sudo -u medcat git pull && sudo systemctl restart medcat` | Обновить код из GitHub |
| `sudo nginx -t && sudo systemctl reload nginx` | Перезагрузить веб-сервер |
| `sudo certbot renew` | Обновить SSL вручную (обычно сам обновляется) |

## 📦 Сохранение проектов

Все данные (проекты, глоссарий, TM) хранятся в:
```
/opt/med-translation/backend/data/state.json
```

Beget делает **автоматические ежедневные бэкапы всего сервера** — если что-то пойдёт не так, можно откатиться на сутки назад через панель Beget.

Хотите ручной бэкап?
```bash
cp /opt/med-translation/backend/data/state.json ~/state-$(date +%Y%m%d).json
```

## 🛡️ Безопасность — рекомендации после установки

### 1. Сменить SSH-пароль на ключ (рекомендую)

Сгенерируйте ключ на своей машине Windows (PowerShell):
```bash
ssh-keygen -t ed25519
```
Скопируйте на сервер:
```bash
type $env:USERPROFILE\.ssh\id_ed25519.pub | ssh root@IP_СЕРВЕРА "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```
Отключите парольный вход в `/etc/ssh/sshd_config`:
```
PasswordAuthentication no
```
```bash
sudo systemctl restart ssh
```

### 2. Создать non-root пользователя для админских задач

```bash
adduser admin
usermod -aG sudo admin
```
Дальше заходите как `ssh admin@IP`, а не root.

### 3. Сменить SSH-порт (опционально)

В `/etc/ssh/sshd_config`:
```
Port 2222
```
Не забудьте открыть в firewall: `sudo ufw allow 2222/tcp`

## ❌ Если что-то сломалось

### Сайт не открывается
```bash
sudo systemctl status medcat    # работает ли приложение?
sudo systemctl status nginx     # работает ли веб-сервер?
sudo journalctl -u medcat -n 50 # последние логи
```

### Не выдался SSL-сертификат
Проверьте, что домен действительно указывает на сервер:
```bash
nslookup med.вашдомен.ru
```
Если IP правильный — запустите вручную:
```bash
sudo certbot --nginx -d med.вашдомен.ru
```

### Закончилось место на диске
```bash
df -h           # сколько свободно
du -sh /opt/*   # что съело
```

### Сервер тормозит
```bash
htop            # посмотреть CPU/RAM (Ctrl+C для выхода)
```
Если RAM забит — апгрейд тарифа в панели Beget (можно «на лету», без переустановки).

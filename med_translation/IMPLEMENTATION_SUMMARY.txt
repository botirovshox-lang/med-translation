================================================================================
USER WORKFLOW
================================================================================

Standard Translation Cycle:

1. Select Segment (✓ button)
   → Displays in right panel

2. Translate (▶️ GPT or 🌐 Google)
   → target_text updated
   → status='translated'

3. QA Check (✓ button)
   → Runs quality checks
   → status='qa_done' or 'needs_review'

4. Review & Edit (in Target field)
   → Auto-saves to database
   → Can re-run QA if needed

5. Confirm (✅ button)
   → status='confirmed'
   → Added to Translation Memory
   → Ready for export

Alternative - Clear (❌ button):
   → Removes translation
   → status='new'
   → Ready for retry

================================================================================
SUPPORT & TROUBLESHOOTING
================================================================================

Issue: Buttons don't appear in table
Solution: Clear browser cache, refresh (Ctrl+F5)

Issue: Translation not saving after confirm
Solution: Check database is writable, verify target field has text

Issue: Google Translate "API not found"
Solution: Enable Cloud Translation API in Google Cloud, wait 5-10 min

Issue: Streamlit "port already in use"
Solution: Kill process or change port: streamlit run app.py --server.port 8502

For more issues, see RELEASE_NOTES.md section "Common Issues & Solutions"

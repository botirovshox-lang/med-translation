# Развёртывание Medical CAT Translator на Render.com + домен с Beget

**Время:** примерно 15 минут.
**Стоимость:** 0 ₽ (бесплатный тариф Render).

---

## ✅ Что получится в итоге

`https://med.вашдомен.ru` → ваш React-дизайн + FastAPI-бэкенд, HTTPS включён, SSL-сертификат продлевается автоматически. Любой `git push origin main` запускает автоматическое обновление сайта.

---

## Шаг 1. Регистрация на Render.com (2 минуты)

1. Откройте https://render.com → кнопка **Get Started**
2. Войдите через **GitHub** (он у вас уже привязан к репозиторию `botirovshox-lang/med-translation`)
3. Разрешите Render читать ваш репозиторий

---

## Шаг 2. Создание сервиса (3 минуты)

1. В панели Render: **New +** → **Blueprint**
2. Выберите репозиторий `botirovshox-lang/med-translation`
3. Render автоматически найдёт файл `render.yaml` и предложит создать сервис `med-translation`
4. Нажмите **Apply** — начнётся автоматическая сборка (около 5 минут)
5. По окончании Render выдаст адрес вида `https://med-translation-XXXX.onrender.com` — приложение уже работает

---

## Шаг 3. Переменные окружения (2 минуты)

В панели Render: ваш сервис → **Environment** → добавьте:

| Переменная | Значение |
|---|---|
| `APP_PASSWORD` | Ваш пароль для входа в приложение |
| `OPENAI_API_KEY` | `sk-...` (если хотите реальные переводы через GPT; без него будут демо-переводы) |
| `GOOGLE_TRANSLATE_API_KEY` | Ключ Google Translate (по желанию) |

После сохранения сервис перезапустится автоматически.

---

## Шаг 4. Привязка домена с Beget (5 минут)

### 4.1. В Render

Ваш сервис → **Settings** → **Custom Domains** → **Add Custom Domain** → введите `med.вашдомен.ru`

Render покажет CNAME-запись, которую нужно добавить, например:
```
CNAME: med-translation-XXXX.onrender.com
```

### 4.2. В панели Beget

1. Зайдите в https://cp.beget.com → раздел **Домены и поддомены**
2. Найдите ваш домен → значок шестерёнки → **DNS-записи** (или **Управление зоной**)
3. Нажмите **Добавить запись** и заполните:
   - **Тип:** `CNAME`
   - **Поддомен:** `med`
   - **Значение:** `med-translation-XXXX.onrender.com` (точное значение из Render)
   - **TTL:** 300
4. Сохраните

### 4.3. Дождаться обновления DNS

Распространение DNS занимает от 5 до 30 минут. Проверить можно командой:
```bash
nslookup med.вашдомен.ru
```

Render автоматически выпустит SSL-сертификат, как только увидит правильную CNAME-запись. После этого открывайте `https://med.вашдомен.ru` — всё работает.

## ⚠️ Особенность бесплатного тарифа

После **15 минут простоя** сервис «засыпает». Первый запрос после простоя — около **30 секунд** на пробуждение, далее всё работает быстро.

Если для команды переводчиков такие задержки неприемлемы — апгрейд до тарифа **Starter** стоит **$7 в месяц**, сервис никогда не спит. Переключение делается одной кнопкой в панели Render.

---

## 🔧 Если что-то пойдёт не так

| Проблема | Что делать |
|---|---|
| **Сборка не удалась** (Build failed) | Логи в Render → **Logs**. Чаще всего — забыли запушить последний коммит |
| **502 Bad Gateway** | Сервис ещё не запустился. Подождите минуту, обновите страницу |
| **Домен не работает** | Проверьте `nslookup med.вашдомен.ru` — должен указывать на `*.onrender.com` |
| **Кнопка «Войти» не работает** | Не задана переменная `APP_PASSWORD` — добавьте в Environment |
| **Перевод выдаёт `[GPT demo translation of #N]`** | Не задан `OPENAI_API_KEY` — это ожидаемое поведение (заглушка для демо) |

## Что важно помнить

- **Старая Streamlit-версия** (по адресу `med-translation-v4de7bewm8xabh9btzatk6.streamlit.app`) продолжает работать параллельно. Если на новой версии что-то пойдёт не так — старая останется доступной как запасной вариант.
- **State приложения** (проекты, глоссарий, TM) сохраняется в `backend/data/state.json` на персистентном диске Render — переживает перезапуски.
- **Без API-ключей** OpenAI/Google приложение работает в **демо-режиме**: показывает заглушки вместо реальных переводов. Все кнопки при этом функциональны.

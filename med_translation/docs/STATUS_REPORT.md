## ⚠️ Известные проблемы

### 1. DOCX импорт (Высокий приоритет)
**Проблема:** При импорте DOCX файла загружается только частичное содержимое
- Ожидается: 142+ сегмента
- Получается: неполное количество строк

**Возможные причины:**
- `iter_docx_text_blocks()` пропускает некоторые текстовые блоки
- Таблицы, текстовые боксы или специальные элементы не обрабатываются
- Фильтр `if text:` пропускает пустые или с пробелами строки

**Статус:** Требуется расследование  
**Действия:** 
- [ ] Проверить структуру DOCX файла
- [ ] Расширить `iter_docx_text_blocks()` для всех типов контента
- [ ] Добавить логирование процесса импорта

### 2. TM% display для новых переводов
**Проблема:** TM% не отображается сразу после новых переводов
- Возможно, fuzzy matching срабатывает не для всех случаев
- Может быть timing issue с сохранением в БД

**Статус:** Низкий приоритет  
**Действия:**
- [ ] Добавить логирование в find_tm_suggestion()
- [ ] Проверить timing сохранения в БД
- [ ] Оптимизировать fuzzy matching

## 🔧 Архитектура

### Поток данных
```
DOCX файл
    ↓ (import_docx_project)
SQLite segments table (142+ records)
    ↓ (st.session_state)
Streamlit UI
    ├─ Левая панель (таблица)
    │   ├─ Поиск/фильтрация
    │   ├─ Пагинация (20/page)
    │   └─ Кнопки действий
    └─ Правая панель
        ├─ Исходный текст
        ├─ Вкладка TM
        ├─ Вкладка QA
        └─ Вкладка Back-check
```

### Модули
| Модуль | Роль |
|--------|------|
| `app_v55.py` | Streamlit UI (основное приложение) |
| `pipeline.py` | Функции обработки (translate, QA, back-check) |
| `docx_cat.py` | Импорт/экспорт DOCX |
| `db.py` | SQLite операции |
| `tm.py` | Поиск в Translation Memory |
| `google_translate.py` | Google Cloud Translate API |
| `openai_client.py` | OpenAI API wrapper |
| `schemas.py` | Pydantic модели для QA отчётов |
| `prompts.py` | LLM prompts для перевода и QA |
